# Security Policy

Thank you for helping keep LiPy secure.

If you discover a security vulnerability, please report it responsibly so it can be investigated and fixed before public disclosure.

---

## Reporting a Vulnerability

Please **do not open a public GitHub Issue** for security vulnerabilities.

Instead, report the issue privately by contacting:

**Email:** contact@lipy.app

Please include:

- A description of the vulnerability.
- Steps to reproduce the issue.
- The potential impact.
- Any proof of concept, screenshots, or logs (if applicable).

---

## Response Process

After receiving a report, the project maintainer will:

1. Acknowledge receipt of the report.
2. Investigate the issue.
3. Develop and test a fix.
4. Release the fix.
5. Publicly disclose the vulnerability, if appropriate, after it has been resolved.

---

## Supported Versions

LiPy is currently under active development.

Security fixes are provided for the latest version of the project.

Older versions may not receive security updates.

---

## Scope

Examples of security issues include, but are not limited to:

- Authentication or authorization bypass
- Remote code execution
- SQL injection
- Cross-site scripting (XSS)
- Cross-site request forgery (CSRF)
- Sensitive data exposure
- Secrets accidentally exposed in the repository
- Dependency-related security vulnerabilities

---

## Responsible Disclosure

Please allow a reasonable amount of time for the issue to be investigated and resolved before publicly disclosing the vulnerability.

Responsible disclosure helps protect users while a fix is being prepared.

Thank you for helping improve the security of LiPy.
