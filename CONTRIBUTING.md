# Contributing to LiPy

Thank you for your interest in contributing to LiPy.

LiPy is an open-source project focused on Odia handwriting recognition, dataset collection, and machine learning. Every contribution, whether code, documentation, design, or bug reports, helps improve the project.

---

## Before You Start

Please:

- Read the README.
- Search existing Issues before opening a new one.
- Open an Issue before working on large features or significant changes.

Small fixes such as typo corrections or documentation improvements do not require an Issue.

---

## Reporting Bugs

Use the **Bug Report** issue template.

Include:

- Steps to reproduce
- Expected behaviour
- Actual behaviour
- Screenshots or logs (if applicable)
- Browser and operating system

---

## Suggesting Features

Use the **Feature Request** template.

Explain:

- The problem
- Your proposed solution
- Why it would benefit the project

---

## Development Workflow

1. Fork the repository.
2. Create a new branch.

```
git checkout -b feature/my-feature
```

3. Make your changes.
4. Test your changes.
5. Commit using clear commit messages.
6. Push your branch.
7. Open a Pull Request.

---

## Commit Messages

Write clear, descriptive commit messages.

Good examples:

```
feat: add contributor statistics page
fix: preserve lifetime contribution count
docs: improve installation guide
refactor: simplify dataset synchronization
```

Avoid messages like:

```
update
fix
changes
done
final
```

Future you will not remember what "final-final-v2-really-final" meant.

---

## Coding Guidelines

- Keep code simple and readable.
- Follow the existing project structure.
- Avoid unnecessary dependencies.
- Write meaningful variable and function names.
- Remove unused code before submitting.

---

## Pull Requests

Before opening a Pull Request, ensure:

- [ ] The project builds successfully.
- [ ] Existing functionality is not broken.
- [ ] New functionality has been tested.
- [ ] Documentation is updated if required.
- [ ] Your branch is up to date with `main`.

---

## Project Structure

```
backend/
frontend/
scripts/
notebooks/
```

Please place new files in the appropriate directory.

---

## Questions

If you are unsure about something, open a GitHub Discussion instead of an Issue.

---

## Code of Conduct

Be respectful and constructive.

Everyone is here to improve the project and help the community.

---

Thank you for contributing to LiPy.
